# ROADMAP FEELTRIP

## Horizonte realista
Roadmap enfocado en convertir FeelTrip en un producto estable, medible y comercialmente defendible.

## Fase 1: Estabilizacion inmediata
- Hardening de auth, router y DI.
- Sync local honesto para momentos.
- Premium estable con RevenueCat.
- Bookings y notificaciones consistentes.
- `flutter analyze` limpio.

## Fase 2: Release serio
- Backend para pagos y webhooks.
- Smoke tests sistematicos.
- Mejoras de cobertura en core flows.
- Reglas de seguridad y observabilidad reforzadas.
- Search mas robusto y experiencia de diario pulida.

## Fase 3: Crecimiento
- Referral loop y sharing medible.
- Notificaciones segmentadas.
- Dashboard de conversion y retencion reales.
- Experimentacion de onboarding y paywall.

## Fase 4: Escala
- Jobs server-side para sync, fanout y pagos.
- Search dedicado si el volumen lo exige.
- Optimizacion de costos Firebase.
- Moderacion, abuse prevention y BI serio.

## Prioridades de negocio
1. Retencion en el diario y momentos.
2. Conversion a premium.
3. Bookings confiables.
4. Canales B2B y afiliacion.

## Riesgos abiertos
- Mercado Pago aun depende de una evolucion backend.
- El sync actual no es una solucion enterprise.
- La cobertura de tests sigue por debajo de lo deseable.
- Algunas docs historicas siguen describiendo fases previas.

## Regla de roadmap
No marcar una fase como completada hasta que:
- exista validacion tecnica,
- exista smoke test estable,
- y exista evidencia de uso real o KPI asociado.
